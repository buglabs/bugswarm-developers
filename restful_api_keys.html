API Keys

Create and manage your API keys with the RESTful API. Users may hold up to two API keys, a configuration key and a participation key. Use the configuration API key for all methods associated with the Configuration API, including establishing swarms and their resources as well as managing invitations. Use the participation API key for all methods associated with the Participation API, including producing and consuming data within one or many swarms.

Create

  • URL: http://api.bugswarm.net/keys
  • Method: POST
  • Authentication: "username:password"
  • Parameters (Required): None
  • Parameters (Optional): Key type

Each user must possess at least a configuration API key. In addition to the configuration key, a participation key can be generated for use with producing and consuming data on the platform. This method is used to create API keys. If a key already exists, this method will replace that key with a new one. For the participation key, users must specify the key type in the URL. For instance, http://api.bugswarm.net/keys/participation will generate the participation key, while http://api.bugswarm.net/keys will generate the configuration key by default. The configuration key may also be created by specifying the key type in the URL.

Usage: 
curl -X POST --user "username:password" http://api.bugswarm.net/keys
curl -X POST --user "username:password" http://api.bugswarm.net/keys/configuration
curl -X POST --user "username:password" http://api.bugswarm.net/keys/participation
Returns: The created key in JSON format. 
{
  "apikey": "911cbc244827dba9c6dbdd09aff3e5e8fe5eee02",
  "type": "configuration"
}

List

  • URL: http://api.bugswarm.net/keys
  • Method: GET
  • Authentication: "username:password"
  • Parameters (Required): None
  • Parameters (Optional): Key type

A user's API key must be passed as a header when using the methods in the BUGswarm API as a form of account authorization. This method will list the API keys associated with the user's account. Users may specify which key they would like to retrieve through the URL. For instance, http://api.bugswarm.net/keys/configuration will retrieve the configuration key, while http://api.bugswarm.net/keys/participation will retrieve the participation key. If no key type is specified in the URL, all keys will be listed.

Usage: 
curl -X GET --user "username:password" http://api.bugswarm.net/keys
curl -X GET --user "username:password" http://api.bugswarm.net/keys/configuration
curl -X GET --user "username:password" http://api.bugswarm.net/keys/participation
Returns: The desired key(s) and their relative information in JSON format.
