RESTful API

Configure swarms and their resources through the use of HTTP requests. Be sure to check out the Tools page for quick and easy ways to jump right into using this API.

API Keys

API keys are used as credentials when interacting with BUGswarm.

Method
Description
Create (POST)

Create an API key. Users can hold up to three API keys, a master key, a producer key, and a consumer key. Each key holds a different set of permissions regarding swarm and resource configuration.

List (GET)

List your API keys. Keys are listed as JSON.

User Resources

Resources belong to a given user account and may be created, updated, and destroyed.

Method
Description
Create (POST)

Create a resource. POST a JSON file containing information about the resource to be created.

Update (PUT)

Update an already existing resource. PUT a JSON file containing the information to update.

Destroy (DELETE)

Destroy a resource. Remove a resource from your user account.

List (GET)

List all of the resources in your user account. Resources listed as JSON.

Get Resource Info (GET)

Get the information relating to a specific resource. Information presented as JSON.

List Swarms With Resource (GET)

List the swarms that a specific resource is a member of. Swarms listed as JSON.

Swarms

Swarms are owned by a single user but may contain the resources of many users.

Method
Description
Create (POST)

Create a swarm. POST a JSON file containing information about the swarm to be created.

Update (PUT)

Update an already existing swarm. PUT a JSON file containing the information to update.

Destroy (DELETE)

Destroy a swarm. Remove a swarm from your user account.

List (GET)

List all of the swarms in your user account. Swarms listed as JSON.

Get Swarm Info (GET)

Get the information relating to a specific swarm, including it's contained resources. Information presented as JSON.

Swarm Resources

Resources can be added and removed from a swarm. Additionally, all of the resources in a swarm may be listed.

Method
Description
Add (POST)

Add a resource to a swarm. POST a JSON file containing the swarm-relevant information regarding the resource to be added.

Remove (DELETE)

Remove a resource from a swarm. Send a JSON file containing swarm-relevant information regarding the resource to remove.

List (GET)

List the resources in a swarm. Users may choose to list all resources, only consuming resources, or only producing resources.
