Swarms

Manage your swarms with the REST API.

Create

  • URL:http://api.bugswarm.net/swarms
  • Method:POST
  • Headers:
    "x-bugswarmapikey: ${API_KEY}",
    "content-type: application/json"
  • Parameters (Required):None
  • Parameters (Optional):None

Each swarm is created by and belongs to a given user, however, many user's resources may be members of a swarm at the swarm owner's discretion. Users can use this method to create a swarm. The method passes a JSON file containing information relevant to the creation of a swarm. Once a swarm is created, users can then addresources to it.

Body:The "name" and "description" fields are required. The "public" field is optional. 
{
    "name": "My Swarm",
    "description": "My Swarm Description",
    "public": false
}
Usage:Create a JSON file with the name create_swarm.jsonand matching the above format. Then execute: 
curl -X POST -d@create_swarm.json --header "x-bugswarmapikey: 911cbc244827dba9c6dbdd09aff3e5e8fe5eee02" --header "content-type: application/json" http://api.bugswarm.net/swarms
Returns:The created swarm and its unique ID in JSON format. 
{
    "swarm":"a16fb1d896817c58d4f4be9dea51e74b86519579"
}

Update

  • URL:http://api.bugswarm.net/swarms/SWARM_ID
  • Method:PUT
  • Headers:
    "x-bugswarmapikey: ${API_KEY}",
    "content-type: application/json"
  • Parameters (Required):None
  • Parameters (Optional):None

Swarms belonging to a given user can have some of their information updated after creation. Updating a swarm involves passing a JSON file containing the fields that the user wishes to update.

Body:All fields are optional. 
{
    "description": "My Swarm description updated",
    "public": true
}
Usage:Create a JSON file with the name update_swarm.jsonand matching the above format. Then execute: 
curl -X PUT -d@update_swarm.json --header "x-bugswarmapikey: 911cbc244827dba9c6dbdd09aff3e5e8fe5eee02" --header "content-type: application/json" http://api.bugswarm.net/swarms/a16fb1d896817c58d4f4be9dea51e74b86519579
Returns:HTTP Response Codes

Destroy

  • URL:http://api.bugswarm.net/swarms/SWARM_ID
  • Method:DELETE
  • Headers:"x-bugswarmapikey: ${API_KEY}"
  • Parameters (Required):None
  • Parameters (Optional):None

Destroy a swarm. Destroying a swarm removes it completely from the user account. The method uses the unique swarm id specified in the URL to determine which swarm to destroy.

Usage:
curl -X DELETE --header "x-bugswarmapikey: 911cbc244827dba9c6dbdd09aff3e5e8fe5eee02" http://api.bugswarm.net/swarms/a16fb1d896817c58d4f4be9dea51e74b86519579
Returns:HTTP Response Codes

List

  • URL:http://api.bugswarm.net/swarms
  • Method:GET
  • Headers:
    "x-bugswarmapikey: ${API_KEY}"
  • Parameters (Required):None
  • Parameters (Optional):Resource type

List all of the swarms owned by the user. Each listed swarm will also contain the resources that are a member of that swarm.

Usage:
curl -X GET --header "x-bugswarmapikey: 911cbc244827dba9c6dbdd09aff3e5e8fe5eee02" http://api.bugswarm.net/swarms
Returns:A list of swarms in JSON format.

Get Swarm Info

  • URL:http://api.bugswarm.net/swarms/${SWARM_ID}
  • Method:GET
  • Headers:
    "x-bugswarmapikey: ${API_KEY}"
  • Parameters (Required):None
  • Parameters (Optional):None

Swarms in a user's carry with them a list of information regarding the swarm in addition to representations of the resources they contain. This method retrieves that information for a given swarm and prints it in JSON format. The ID of the swarm to retrieve the information from is given in the URL.

Usage:
curl -X GET --header "x-bugswarmapikey: 911cbc244827dba9c6dbdd09aff3e5e8fe5eee02" http://api.bugswarm.net/swarms/a16fb1d896817c58d4f4be9dea51e74b86519579
