REST API

Configure Swarms and their Resources through the use of HTTP requests.

API Keys

API Keys are used as credentials when interacting with BUGswarm.

Method
Description
Create (POST)

Create an API Key. Users can hold up to 3 API Keys, a master key, producer key, and consumer key. Each key holds a different set of permissions.

List (GET)

List your API keys. Keys are listed as JSON.

User Resources

Resources belong to a given User account and may be created, updated, and destroyed.

Method
Description
Create (POST)

Create a Resource. POST a JSON file containing information about the Resource to be created.

Update (PUT)

Update an already existing Resource. PUT a JSON file containing the information to update.

Destroy (DELETE)

Destroy a Resource. Remove a Resource from your User account.

List (GET)

List all of the Resources in your User account. Resources listed as JSON.

Get Resource Info (GET)

Get the information of a specific Resource. Information presented as JSON.

List Swarms With Resource (GET)

List the Swarms that a specific Resource belongs to. Swarms listed as JSON.

Swarms

Swarms are owned by a User but may contain the Resources of many Users.

Method
Description
Create (POST)

Create a Swarm. POST a JSON file containing information about the Swarm to be created.

Update (PUT)

Update an already existing Swarm. PUT a JSON file containing the information to update.

Destroy (DELETE)

Destroy a Swarm. Remove a Swarm from your User account.

List (GET)

List all of the Swarms in your User account. Swarms listed as JSON.

Get Swarm Info (GET)

Get the information of a specific Swarm, including it's contained Resources. Information presented as JSON.

Swarm Resources

Resources can be added and removed from a Swarm. Additionally, all of the Resources in a Swarm may be listed.

Method
Description
Add (POST)

Add a Resource to a Swarm. POST a JSON file containing information about the Resource to be added that pertains to being a member of a Swarm.

Remove (DELETE)

Remove a Resource from a Swarm. Send a JSON file containing information about the Resource to remove.

List (GET)

List the Resources in a Swarm. Users may choose to list all Resources, only Consumers, or only Producers.
