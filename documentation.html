API Libraries

BUGswarm makes use of two primary APIs for configuring and participating in swarms:

REST API

BUGswarm uses a REST (Representational State Transfer) API for configuring swarms and their resources. The API consists of several HTTP GET, POST, DELETE, and PUT requests through the use of curl. The REST API is not a streaming API. For each method call, a connection is opened, HTTP requests are sent, HTTP responses are returned, and the connection is closed. This API is used for creating and destroying swarms and resources, adding and removing resources from swarms, and generating a user's API keys.

Streaming API

The BUGswarm Streaming API is used by users and resources to participate in a configured swarm. The Streaming API also uses HTTP requests, however, it does not follow REST guidelines. In the Streaming API, connections are opened and remain open while HTTP requests are sent in succession. This API is used for producing and consuming information from a swarm.
